这篇教程适合谁

如果你和我一样，对笔记系统没有太高要求，只是想把 CiteBox 里整理出来的一些结果顺手保存到 Wolai，这个配置方式就很合适。它不是一套复杂的知识管理方案，而是一个稳定、够用、维护成本低的同步出口。

Why This Setup
先把最小可用链路跑通：拿到 Token，填上页面 ID，确认能写入。

后面再决定是否扩展流程，不需要一开始就把笔记系统设计得很重。

Quick Start

先准备这 3 个值

在 CiteBox 的 Wolai 配置里，你只需要填写三项内容。最容易的是 OpenAPI 地址，最常抄错的是目标页面 ID，最麻烦的是 Wolai Token。

01

Wolai Token

通过你创建的 Wolai 应用换取，用来让 CiteBox 调用 Wolai OpenAPI。

02

OpenAPI 地址

最简单，直接填写 https://openapi.wolai.com。

03

目标页面 ID

也就是你希望 CiteBox 往哪个 Wolai 页面下面继续插入内容。

界面里这一项叫 OpenAPI 地址。如果你平时口头说的是“OpenAI 地址”，这里实际要填的也是 Wolai 的 OpenAPI 网关，而不是 OpenAI 官方地址。
Target Page

目标页面 ID 怎么取

这一步不复杂。打开你想接收笔记的 Wolai 页面，点击页面右上角复制当前页面链接，然后取链接最后一段即可。

例如页面链接 https://www.wolai.com/wolai/k1Qgi1J2L9vWBQkkJz8j82，最后这段 k1Qgi1J2L9vWBQkkJz8j82 就是你要填写的目标页面 ID。
这个字段对应 CiteBox 内部的 parent_block_id。填对了，后续测试 Token 时也会顺便验证当前 Token 是否有权访问这个页面。
Token

Wolai Token 是最难拿到的部分

要先在 Wolai 里创建一个应用，拿到 App ID 和 App Secret，然后再用它们调用 Wolai OpenAPI 的创建 Token 接口。

1

打开个人设置里的应用设置

先点击左上角自己的头像，进入个人设置，然后选择“应用设置”，再点击“创建和管理应用”。

2

创建一个新应用

进入应用管理页面后，点击“创建新应用”。

3

把权限全部勾选上

为了先把集成跑通，权限可以直接全勾。后面如果你想做更细的权限收缩，再回头精简。

这篇教程的目标是先确认 CiteBox 能创建页面、写入内容和完成测试，不建议在第一次配置时过早缩权限。
4

记下 App ID 和 App Secret

应用创建成功后，记住这两个值。下一步在终端里换取 Token 时就要用到它们。

Terminal

在终端里换取 Wolai Token

下面这段命令直接调用 Wolai 的创建 Token 接口。你只需要把自己的应用 ID 和应用密钥填进去即可。

接口参考：Wolai API / 创建 Token。当前接口地址是 https://openapi.wolai.com/v1/token。
bash
curl --location --request POST 'https://openapi.wolai.com/v1/token' \
--header 'Content-Type: application/json' \
--data-raw '{
    "appId": "填入你的应用ID",
    "appSecret": "填入你的应用密钥"
}'

如果你在窄屏上看这页，直接点上面的“复制命令”会更省事；这个代码块现在也支持自动换行。

执行成功后，会返回一段 JSON。把其中可以直接调用 OpenAPI 的 Token 复制出来，填回 CiteBox 的 Wolai Token 输入框即可。
Verification

最后一步：测试通过，再保存配置

拿到 Token 后，回到 CiteBox 设置页，把三项都填好，先点“测试 Token”。测试通过后，再点击“保存 Wolai 配置”。

Wolai Token：填你刚刚在终端里换到的 Token。
OpenAPI 地址：填写 https://openapi.wolai.com。
目标页面 ID：填写页面链接最后一段。
先点“测试 Token”，确认 Token 可用且当前页面可访问。
测试通过后再保存；如果你想再稳一点，可以额外点一次“插入测试页面”。
CiteBox 当前的测试逻辑会校验目标页面访问权限，所以“测试 Token”通过，基本就说明这套配置已经通了。

这样配置完，就可以开始用了

对于“只想把结果稳稳存进笔记里”这个目标来说，这套 Wolai 配置已经足够。先把链路跑通，比一开始追求完美配置更重要。
